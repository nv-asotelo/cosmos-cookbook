**Access:** BYO (Bring Your Own) — no public dataset is distributed.
Motion trajectories are generated inside **NVIDIA Omniverse / Isaac Sim** from a small
set of human manipulation demonstrations.

This recipe is a reference workflow, not a pull-and-run benchmark.
See [build.nvidia.com/nvidia/isaac-gr00t-synthetic-manipulation](https://build.nvidia.com/nvidia/isaac-gr00t-synthetic-manipulation)
for the interactive demo and hosted API.

## Compute Requirements

- **Isaac Sim / Omniverse**: Requires NVIDIA Omniverse installed locally (not available on Brev cloud GPU instances without special setup)
- **Cosmos Transfer 1 inference**: 1x H100-80GB (>= 70 GB VRAM free)
- **Full pipeline**: Isaac Sim simulation node + 1x H100-80GB for video generation

> **Note:** Isaac Sim requires a desktop or workstation Omniverse installation.
> The Brev brev.yaml for this recipe provisions the Cosmos Transfer 1 inference
> environment only. Isaac Sim / Omniverse must be run separately on a compatible
> workstation and used to export motion data before running demo.sh.

## Execution

Headless execution for the Cosmos Transfer 1 video generation step only.
Isaac Sim simulation is out of scope for headless cloud execution.

### Pre-flight

```bash
nvidia-smi
```

Confirm H100-80GB is visible with >= 70 GB VRAM free.

### Run

```bash
# Set path to your Isaac Sim-exported motion video (BYO)
export BYO_VIDEO=/path/to/your/control_video.mp4

bash deploy/transfer1/gr00t-mimic/demo.sh
```

### What it runs

1. Validates GPU VRAM (>= 70 GB required for Cosmos Transfer 1)
2. Clones `cosmos-transfer1` and installs dependencies via `uv`
3. Accepts a user-provided control video exported from Isaac Sim
4. Runs Cosmos Transfer 1 to generate a photorealistic manipulation video
5. Writes timing metrics to `/tmp/gr00t_mimic_results.json`

### Success criteria

- `CUDA available: True` during setup
- `gr00t_mimic_results.json` contains `"status": "success"`
- Output video exists at `/tmp/gr00t_mimic_output/output.mp4`
